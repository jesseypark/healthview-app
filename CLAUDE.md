# HealthView — Claude Instructions

## Before starting any work session

Read all three files in `/docs` before touching any code:

- `/docs/PROJECT_MAP.md` — architecture, file locations, how things connect
- `/docs/DECISIONS.md` — non-obvious design choices and the reasoning behind them
- `/docs/HANDOFF.md` — current state: what's done, what's in progress, what's broken

## At the end of each work session

Update the docs to reflect any changes made:

- `PROJECT_MAP.md` — if new files, screens, or services were added
- `DECISIONS.md` — if a non-obvious pattern was introduced or an existing one was changed
- `HANDOFF.md` — update "Current State" and move completed items; note anything left unfinished or newly discovered
